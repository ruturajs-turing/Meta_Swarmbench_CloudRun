Lifecycle

The parent persists. Every child is disposable.

The trainer can disconnect without losing an active run because the control plane, not the terminal session, owns execution.

PARENTone per trainer
PROVISIONT+0 · first sign-inCreate the parent from the current managed image and mount the trainer's persistent workspace.

PAUSEDAfter 15 minutes idleSuspend parent compute to reduce cost. Tasks, runs, and downloads remain on persistent storage.

REFRESHEDAfter 24 hours, when idleDrain active work, recreate from the latest managed image, and remount the preserved workspace.

Refresh and destroy are blocked while children are active. Idle parents pause after 15 minutes; refresh is due after 24 hours; orphan cleanup runs independently of the trainer session.

Guardrails

Security, cleanup, and cost are platform behavior

The trainer cannot bypass these controls from the parent or from task TOML.

Isolation

  • one fresh child per run
  • safe archive extraction
  • tenant-scoped workspaces
  • no provider keys in parent

Authority

  • control-plane quotas
  • managed runner image
  • audited admin mutations
  • server-side TOML validation

Cost control

  • hard child timeout
  • queue over concurrency
  • idle parent pause
  • finalizer plus orphan reconciler

Internal setup

Local Docker mirrors the cloud contract

Probe mode proves transfer, resources, lifecycle, artifacts, and cleanup without model spend. Harbor mode adds the managed runner and Fireworks secret.

# 1. build the maintained Harbor/OpenCode image
docker build -f runtimes/harbor-runner/Dockerfile \
  -t aegisrun/harbor-runner:local ../harbor

# 2. start the control plane and managed runtime
AEGISRUN_HARBOR_RUNTIME_ENABLED=true \
AEGISRUN_HARBOR_RUNNER_IMAGE=aegisrun/harbor-runner:local \
FIREWORKS_API_KEY=<secret> \
docker compose up -d --build

# 3. enter the trainer interface
./cli/runnerctl connect --endpoint http://localhost:18080 \
  --user trainer

Cloud evolution

The runtime changes. The trainer contract does not.

The local Docker adapter proves the behavior. E2B, Daytona, or a self-hosted sandbox provider can replace the execution adapter without changing the task path, TOML, events, or result flow.
